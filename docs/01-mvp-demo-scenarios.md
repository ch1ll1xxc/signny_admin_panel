# MVP Demo Scenarios (для показа заказчику)

## Сценарий 1: Экспонат от source до publish
1. Editor создает экспонат и source RU.
2. Запускает preprocess (`Job: preprocess`).
3. Получает draft, редактирует вручную.
4. Отправляет в review (`on_review`).
5. Curator оставляет комментарий и одобряет (`approved`).
6. Admin публикует через atomic publish (`published`).
7. Система показывает QR-связку и public-видимость.

Критерий успеха:
- Во внешнем контуре видна только опубликованная версия.

## Сценарий 2: Возврат на доработку
1. Curator открывает версию в `on_review`.
2. Оставляет комментарий с причиной и переводит в `needs_revision`.
3. Editor правит контент и возвращает в `on_review`.
4. Curator одобряет.

Критерий успеха:
- В истории сохранены причина, автор и время ревизии.

## Сценарий 3: Отдельная ветка FAQ
1. Editor создает FAQ item и версию ответа.
2. Проходит review/approve lifecycle.
3. Admin публикует FAQ в общий контур.

Критерий успеха:
- FAQ не смешивается с Exhibit-сущностью, но подчиняется тем же governance-правилам.

## Сценарий 4: Аудит и выгрузка
1. Admin открывает audit log и фильтрует по `correlationId`.
2. Открывает analytics (два таба: user/admin).
3. Запускает export, отслеживает async completion, скачивает файл.

Критерий успеха:
- Любое ключевое действие из сценариев 1-3 трассируется в аудите.
