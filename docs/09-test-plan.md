# Программа и методика испытаний АК АС «Сайнни»

## 1. Объект испытаний

Административный контур автоматизированной системы «Сайнни» — веб-приложение управления контентом музейной экспозиции для слабослышащих.

## 2. Цель испытаний

Подтверждение соответствия функциональным (п.5.2) и нефункциональным (п.5.3) требованиям ТЗ.

## 3. Виды испытаний

| Вид | Инструмент | Объём |
|-----|-----------|-------|
| Модульные | Vitest | Бизнес-логика composables и stores |
| Интеграционные | Vitest + mock API | Связь composables ↔ API |
| Функциональные (e2e) | Playwright | 41 тест по требованиям ТЗ |
| Приёмочные | Ручная проверка | По чеклисту п.7.2 ТЗ |

## 4. Автоматизированные e2e-тесты

Файл: `tests/e2e/tt-requirements.spec.ts`

### 5.2.1 Управление карточками экспонатов (5 тестов)
- [ ] Список экспонатов отображается с таблицей
- [ ] Кнопка создания экспоната доступна
- [ ] Диалог создания экспоната открывается
- [ ] Карточка экспоната открывается по клику
- [ ] Фильтры (поиск, экспозиция, статус) доступны

### 5.2.2 Работа с исходным текстом (1 тест)
- [ ] Раздел «Контент версии» доступен на карточке экспоната

### 5.2.3 Предварительная подготовка текста для РЖЯ (1 тест)
- [ ] Кнопка «Запустить предобработку» отображается

### 5.2.4 Ручная проверка и доработка (1 тест)
- [ ] Кнопка «Редактировать» доступна на карточке

### 5.2.5 Управление FAQ (4 теста)
- [ ] Страница FAQ загружается
- [ ] Кнопка добавления FAQ доступна
- [ ] Форма создания FAQ открывается
- [ ] FAQ элементы отображаются (seed data)

### 5.2.6 Подготовка материалов для публичного контура (3 теста)
- [ ] Страница публикации загружается
- [ ] Секция предварительной проверки с кнопкой
- [ ] Секция массовой публикации

### 5.2.7 Жизненный цикл версий контента (3 теста)
- [ ] Очередь модерации отображается
- [ ] Действия модерации доступны (согласовать, на доработку)
- [ ] Таблица экспонатов содержит колонку статуса

### 5.2.8 Комментарии и согласование (1 тест)
- [ ] Поле комментария модератора доступно

### 5.2.9 Публикация (1 тест)
- [ ] Preflight-проверка запускается и показывает результат

### 5.2.10 Журналирование (3 теста)
- [ ] Страница аудита загружается
- [ ] Таблица с заголовками (Время, Действие)
- [ ] Фильтры доступны (2 select)

### 5.2.11 Аналитика и выгрузки (4 теста)
- [ ] Страница аналитики загружается
- [ ] Страница выгрузок загружается
- [ ] 3 кнопки «Скачать CSV» доступны
- [ ] Карточки экспорта: версии, аудит, FAQ

### 5.3.1 Надёжность (1 тест)
- [ ] Дашборд загружается с метриками при mock-данных

### 5.3.2 Производительность (1 тест)
- [ ] Все 10 страниц загружаются менее чем за 3 секунды каждая

### 5.3.3 Информационная безопасность (5 тестов)
- [ ] Страница логина отображается
- [ ] Кнопки входа по ролям (admin, editor, curator)
- [ ] Неавторизованный доступ перенаправляет на логин
- [ ] Авторизация как admin → роль «Администратор»
- [ ] Авторизация как editor → роль «Редактор»

### 5.3.4 Совместимость — навигация (1 тест)
- [ ] Сайдбар содержит все 10 разделов по ТЗ

### Дополнительные тесты (6 тестов)
- [ ] Залы: страница, кнопка создания, таблица
- [ ] Медиатека: страница, таблица, поиск

## 5. Запуск тестов

```bash
# Установка зависимостей (однократно)
npm install
npx playwright install chromium

# Запуск всех e2e-тестов
npx playwright test tests/e2e/tt-requirements.spec.ts

# Запуск с визуальным отчётом
npx playwright test tests/e2e/tt-requirements.spec.ts --reporter=html
npx playwright show-report
```

## 6. Критерии приёмки

Административный контур считается принятым при:
- Все 41 автоматизированный тест проходят (41/41 PASS)
- Отсутствуют критические ошибки сборки (`vite build` без ошибок)
- Все маршруты доступны после авторизации
- Публикация не нарушает работу публичного контура
